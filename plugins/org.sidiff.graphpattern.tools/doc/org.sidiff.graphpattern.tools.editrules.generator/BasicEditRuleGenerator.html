Matching of Edit Rule Generator

Pattern A (Pre) Pattern B (Post) Edit Rule Comment
<<?>> <<?>> <<preserve>>  
<<not>> <<?>> <<forbid (precondition)>> and <<create>> not supported (1)
<<exists>> <<?>> <<preserve>>  
<<?>> <<not>> <<delete>> not supported (1)
<<not>> <<not>> <<forbid>> not supported (1)
<<exists>> <<not>>   not allowed (2)
<<?>> <<exists>> <<preserve>>  
<<not>> <<exists>>   not allowed (2)
<<exists>> <<exists>> <<preserve>> (3)

(1) Currently only mappings are supported that lead to <<preserve>> nodes in the edit rule.

(2) <<not>> means the element is not existing, <<exists>> the element already exists (without further modifications), it follows a mapping is not possible.

(3) Overlapping of pre- and post-condition graph, which needs a context node in the LHS.
