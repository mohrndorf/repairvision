Example: [62] [A Rule-Based System for Assessing Consistency Between UML]

			----------------------------------------------------------------------------------------
			20 - Use Case Diagram and Class Diagram
			----------------------------------------------------------------------------------------
Rule 141: 	The name of a use  case must include a verb and a noun.
Rule 142: 	The noun of the use  case's name should correspond to the name of one class in the class 
			diagram. In other words, for each use  case U in the class diagram, there should be a 
			class  C belonging to the class  diagram, so that U.name equals C.name.
			----------------------------------------------------------------------------------------
			Classifier
				self.UseCase->exists(us: Usecase, c: Class, x: Integer, y: Integer | y > x
				and us.name.toUpper.substring(x,y)=c.name.toUpper)
			----------------------------------------------------------------------------------------
Rule 143: 	The verb of the use  case's name should correspond to an operation of a class in the 
			class diagram. In other words, for each use  case U there should be a class C that 
			contains an operation Operation(x) so that U.name contains C.Operation(x).
			----------------------------------------------------------------------------------------
			Classifier
				self.UseCase->exists(us: Usecase, c: Class, x: Integer, y: Integer | y > x
				and us.name.toUpper.substring(x,y)=c.operation.toUpper)
			----------------------------------------------------------------------------------------
